CreativeSpark Identity Verification Hub
A secure, serverless biometric identity verification pipeline built on AWS. This project demonstrates an event-driven architecture using AWS Lambda, S3, DynamoDB, and Amazon Rekognition to automate user identity checks.

Architecture Overview
Upload Subservice: An HTTP API that accepts user selfies and government-issued IDs, vaults them securely into S3, and triggers an asynchronous verification worker.

Verification Worker: A background process that leverages Amazon Rekognition to perform facial comparison between the uploaded ID and the user selfie, updating the status in a DynamoDB ledger.

Security: This project implements production-grade security by utilizing environment-level variable management, avoiding hardcoded secrets in the source code.

Key Technologies
Compute: AWS Lambda (Python 3.11)

Storage: Amazon S3 & Amazon DynamoDB

AI/ML: Amazon Rekognition

Infrastructure: AWS Serverless Application Model (SAM)

Deployment Instructions
Prerequisites
Ensure you have the AWS CLI and AWS SAM CLI installed.

Configure your local AWS credentials via aws configure.

Environment Setup
For security, this project requires the following environment variables to be set in your AWS Lambda configuration:

AUTH_TOKEN: Your custom secret token for API authorization.

S3_BUCKET_NAME: The name of your secure S3 storage bucket.

DYNAMODB_TABLE_NAME: The name of your DynamoDB ledger table.

VERIFY_WORKER_FUNCTION: The ARN or name of the worker Lambda function.

Frontend Configuration
To connect the frontend to your backend:

Open frontend/index.html.

Locate the const API_URL and const SESSION_TOKEN lines at the top of the script.

Update API_URL with the endpoint generated by your AWS SAM deployment.

Update SESSION_TOKEN to match the AUTH_TOKEN environment variable you set in your AWS Lambda console.

Deployment
To deploy this architecture to your AWS account, run:

Bash
sam build
sam deploy --guided
Security & Compliance
This repository is configured to exclude sensitive files. Please ensure you create a .env file for local development and add it to your .gitignore to prevent the leakage of your API keys or tokens.
